# ORD Database Setup Guide

This guide walks you through setting up an Open Reaction Database (ORD) stored in Google Sheets and exposed as a REST API.

## Prerequisites

- Python 3.8+
- Google Cloud Console account
- Git (for cloning ORD data)

## Step 1: Parse ORD Data

The ORD data has been downloaded and parsing scripts are ready.

```bash
# Parse a few sample files (adjust max_files as needed)
python parse_ord.py ord-data/data/00 ord_reactions.csv 5

# Parse all files in a directory
python parse_ord.py ord-data/data ord_reactions.csv
```

## Step 2: Set Up Google Sheets API

### 2.1 Create Google Cloud Project
1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project or select existing one
3. Enable billing if required

### 2.2 Enable Google Sheets API
1. In Cloud Console, go to "APIs & Services" > "Library"
2. Search for "Google Sheets API"
3. Click "Enable"

### 2.3 Create Service Account
1. Go to "APIs & Services" > "Credentials"
2. Click "Create Credentials" > "Service Account"
3. Fill in service account details
4. Grant roles: "Editor" for Google Sheets
5. Create and download JSON key file as `credentials.json`
6. Place `credentials.json` in this directory

### 2.4 Share Google Drive Folder (Optional)
1. Create a folder in Google Drive for your spreadsheets
2. Share it with the service account email (from credentials.json)
3. Grant "Editor" access

## Step 3: Upload Data to Google Sheets

```bash
# Upload CSV to new Google Sheet
python upload_to_sheets.py ord_reactions.csv "ORD Reactions Database"

# Upload and share with specific email
python upload_to_sheets.py ord_reactions.csv "ORD Database" your.email@example.com
```

This will:
- Create a new Google Sheet
- Upload all reaction data
- Generate an Apps Script file for API access

## Step 4: Deploy Apps Script API

1. Open the created Google Sheet
2. Go to **Extensions > Apps Script**
3. Delete any default code
4. Copy and paste the contents of `api_script.js` (generated by upload script)
5. Click **Save** (give it a name like "ORD API")
6. Click **Deploy > New deployment**
7. Configure:
   - **Type**: Web app
   - **Description**: ORD Reactions API
   - **Execute as**: Me
   - **Who has access**: Anyone
8. Click **Deploy**
9. **Copy the deployment URL** - this is your API endpoint!

## Step 5: Test the API

```bash
# Get all reactions
curl "YOUR_DEPLOYMENT_URL"

# Search for aldol reactions
curl "YOUR_DEPLOYMENT_URL?query=aldol"

# Search for specific SMILES
curl "YOUR_DEPLOYMENT_URL?query=CCO"
```

## Step 6: Integrate into React App

Add this to your React component:

```javascript
const ORD_API_URL = 'YOUR_DEPLOYMENT_URL';

async function searchReactions(query) {
  const response = await fetch(`${ORD_API_URL}?query=${encodeURIComponent(query)}`);
  const reactions = await response.json();
  return reactions;
}

// Usage
const reactions = await searchReactions('Buchwald');
console.log(reactions);
```

## Troubleshooting

### Authentication Issues
- Ensure `credentials.json` is in the correct directory
- Check that the service account has access to Google Sheets API
- Verify the JSON key file is valid

### API Deployment Issues
- Make sure the Apps Script is deployed as a web app
- Check that "Anyone" has access to the deployment
- Verify the spreadsheet ID in the script matches your sheet

### Data Parsing Issues
- Ensure ord-schema is properly installed
- Check that ORD files are not corrupted
- Verify file paths are correct

## Scaling Considerations

- **Large Datasets**: Split into multiple sheets or use Google BigQuery
- **API Limits**: Apps Script has execution time limits (~6 minutes)
- **Rate Limits**: Implement caching in your React app
- **Updates**: Re-run parsing/upload when ORD data updates

## Security Notes

- The API is publicly accessible - consider authentication for production
- Service account keys should be kept secure
- Monitor Google Cloud billing for API usage
